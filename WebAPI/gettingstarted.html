Getting Started

Test Environment

Create an account on our Trickster West testing website at https://tricksterwest.azurewebsites.net/game/. This site may start up slowly because it uses shared infrastructure.

Visit https://tricksterwest.azurewebsites.net/webbotconfig/ to configure the endpoints for your bot. An empty JSON structure is presented. Replace the game methods you support with the absolute URLs of your endpoint.

Here is an example filled out with endpoints that return random suggestions: 

{
  "Euchre": {
    "card": "https://tricksterbotswest.azurewebsites.net/suggest/random/card",
    "bid": "https://tricksterbotswest.azurewebsites.net/suggest/random/bid"
  },
  "Hearts": {
    "card": "https://tricksterbotswest.azurewebsites.net/suggest/random/card",
    "pass": "https://tricksterbotswest.azurewebsites.net/suggest/random/pass"
  }
}

Creating Endpoints

We strongly recommend building your bots in C# and .NET Framework 4.8 starting with our example project and using the base types it provides.

Note: Our code that calls these endpoints use Newtonsoft.Json for JSON serialization. If you don’t use Newtonsoft.Json in your endpoint, you will see some class fields with different names in your deserialized objects.

All endpoints are called with POST. The POST data is a string of serialized JSON. The objects serialized differ somewhat for each call. See the SuggestXxxxState objects in the API Types Documentation.

Similarly, each endpoint should return a JSON-serialized string of BidBase, Card, Hand, or List<Card> depending on the method called.
